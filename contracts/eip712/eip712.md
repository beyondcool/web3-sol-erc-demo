# EIP-712 开发速查手册

> 合约继承 OpenZeppelin `EIP712` 后的必做步骤，读完即用。

---

## 总览：EIP-712 核心公式（4 步）

```
第 1 步：domainSeparator = keccak256(EIP712Domain 类型编码)
         ← OZ 自动完成（EIP712 构造函数）

第 2 步：typeHash = keccak256("StructName(type1 field1,type2 field2,…)")
         ← 你定义

第 3 步：structHash = keccak256(abi.encode(typeHash, encodeData(fields…)))
         ← 你实现

第 4 步：digest = keccak256(0x19 0x01 || domainSeparator || structHash)
         ← OZ 的 _hashTypedDataV4() 完成
```

**OZ 帮你做了第 1、4 步。你只需要关心第 2 步（定义结构体）和第 3 步（实现 hashStruct）。**

---

## 第一步：继承并初始化 EIP712

```solidity
import "@openzeppelin/contracts/utils/cryptography/EIP712.sol";

// 方式一：内联初始化（推荐，与本目录 EIP712Demo.sol 一致）
contract MyContract is EIP712("MyDApp", "1") {
    // ...
}

// 方式二：构造函数初始化
contract MyContract is EIP712 {
    constructor() EIP712("MyDApp", "1") {
        // ...
    }
}
```

父类利用 `name` 和 `version`，结合 `block.chainid` 和 `address(this)`，自动计算并缓存 `domainSeparator`。

---

## 第二步：定义业务数据结构体

```solidity
/// @notice 用户签名的业务数据结构
struct Note {
    string  content;   // 业务内容
    uint256 nonce;     // 防重放计数器
}
```

> 结构体是用户要在钱包里确认「我签的是什么」的可读数据。可根据业务需要添加任意多个字段（`address`、`uint256`、`string` 等均可）。

---

## 第三步：定义类型哈希（TypeHash）

```solidity
/// @notice 类型哈希：唯一标识这个数据结构
bytes32 public constant NOTE_TYPEHASH = keccak256(
    "Note(string content,uint256 nonce)"
);
```

**⚠️ 拼写规则**：
- 格式：`StructName(type1 field1,type2 field2,…)`
- 字段之间**没有空格**，末尾**没有分号**
- `uint256` 不能简写为 `uint`
- `string`、`bytes`、数组等动态类型直接写类型名
- **字段顺序**必须与 struct 定义完全一致

如果写错，链下签名和链上验签会匹配不上。

---

## 第四步：实现 hashStruct

```solidity
function hashNote(Note calldata note) public pure returns (bytes32) {
    return keccak256(abi.encode(
        NOTE_TYPEHASH,
        keccak256(bytes(note.content)),  // 动态类型 → 先哈希再 encode
        note.nonce                         // 值类型 → 直接 encode
    ));
}
```

### EIP-712 类型编码规则

| 类型 | 编码方式 | 示例 |
|------|---------|------|
| `uint*`, `int*`, `address`, `bool`, `bytes32` | 直接 `abi.encode` | `note.nonce` |
| `string` | `abi.encode(keccak256(bytes(value)))` | `keccak256(bytes(note.content))` |
| `bytes` | `abi.encode(keccak256(value))` | `keccak256(note.rawData)` |
| `T[]`（动态数组） | `abi.encode(keccak256(abi.encodePacked(elements)))` | 先折叠再哈希 |
| 嵌套结构体 | 递归调用 `hashNested()`，然后把 `bytes32` 传入 | `hashSubStruct(sub)` |

> **OZ 不替你编码 struct——它不知道你的 struct 长什么样。你必须自己写 hashStruct。**

---

## 第五步：生成最终摘要（Digest）

```solidity
/// @notice 生成 EIP-712 签名摘要
function getDigest(Note calldata note) public view returns (bytes32) {
    return _hashTypedDataV4(hashNote(note));
}
```

`_hashTypedDataV4` 内部等价于：

```solidity
keccak256(abi.encodePacked(
    hex"19", hex"01",
    _domainSeparatorV4(),  // OZ 自动计算
    structHash
))
```

## 第六步：验签逻辑

### 方式一：裸 ecrecover（简单场景，与 EIP712Demo.sol 一致）

```solidity
function signNote(Note calldata note, uint8 v, bytes32 r, bytes32 s) external {
    bytes32 digest = _hashTypedDataV4(hashNote(note));
    address recovered = ecrecover(digest, v, r, s);
    require(recovered == msg.sender, "签名无效");
    // ... 执行业务逻辑
}
```

### 方式二：ECDSA.recover（推荐生产使用）

```solidity
import "@openzeppelin/contracts/utils/cryptography/ECDSA.sol";

function signNote(Note calldata note, bytes calldata signature) external {
    bytes32 digest = _hashTypedDataV4(hashNote(note));
    address recovered = ECDSA.recover(digest, signature);
    require(recovered == msg.sender, "签名无效");
    // ... 执行业务逻辑
}
```

**推荐用 `ECDSA.recover`**，它内部做了签名可锻性（malleability）防护。

---

## 第七步：防重放攻击

> EIP-712 只负责签名格式，**防重放是你自己的责任**。

### 方案一：自定义 nonce 映射（与 EIP712Demo.sol 一致）

```solidity
mapping(address => mapping(uint256 => bool)) public usedNonces;

function signNote(Note calldata note, ...) external {
    require(!usedNonces[msg.sender][note.nonce], "Nonce 已被使用");
    // ... 验证签名 ...
    usedNonces[msg.sender][note.nonce] = true;
}
```

### 方案二：OZ 的 Nonces 合约（推荐）

```solidity
import "@openzeppelin/contracts/utils/Nonces.sol";

contract MyContract is EIP712, Nonces {
    // 合约自动维护每个地址的 nonce 计数器
    // 签名时调用 _useNonce(signer) 即可
}
```

## 补充模式：只读验签 vs 签名+改状态

如果业务需要支持中继器（relayer）或 gasless 场景，可以把只读验签和状态修改分拆为两个函数：

```solidity
// 🔍 只读验签（不消耗 gas，供后端/中继器预检查）
function verify(
    Note calldata note,
    address expectedSigner,
    uint8 v, bytes32 r, bytes32 s
) external view returns (bool) {
    bytes32 digest = _hashTypedDataV4(hashNote(note));
    return ecrecover(digest, v, r, s) == expectedSigner;
}

// ✍️ 签名 + 改状态（消耗 gas）
function signNote(Note calldata note, uint8 v, bytes32 r, bytes32 s) external {
    bytes32 digest = _hashTypedDataV4(hashNote(note));
    address recovered = ecrecover(digest, v, r, s);
    require(recovered == msg.sender, "签名无效");
    // ... 修改状态 ...
}
```

## 完整最小模板

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/utils/cryptography/EIP712.sol";
import "@openzeppelin/contracts/utils/cryptography/ECDSA.sol";

contract MyContract is EIP712("MyDApp", "1") {
    // ── 数据结构（可根据业务添加更多字段） ──
    struct Note {
        string  content;
        uint256 nonce;
    }

    // ── 类型哈希 ──
    bytes32 public constant NOTE_TYPEHASH = keccak256(
        "Note(string content,uint256 nonce)"
    );

    // ── 状态 ──
    mapping(address => mapping(uint256 => bool)) public usedNonces;

    // ── hashStruct ──
    function hashNote(Note calldata note) public pure returns (bytes32) {
        return keccak256(abi.encode(
            NOTE_TYPEHASH,
            keccak256(bytes(note.content)),
            note.nonce
        ));
    }

    // ── 获取摘要（可选暴露） ──
    function getDigest(Note calldata note) public view returns (bytes32) {
        return _hashTypedDataV4(hashNote(note));
    }

    // ── 签名 + 改状态 ──
    function signNote(
        Note calldata note,
        uint8 v,
        bytes32 r,
        bytes32 s
    ) external {
        require(!usedNonces[msg.sender][note.nonce], "Nonce used");

        bytes32 digest = _hashTypedDataV4(hashNote(note));
        address recovered = ecrecover(digest, v, r, s);
        require(recovered == msg.sender, "Invalid signature");

        usedNonces[msg.sender][note.nonce] = true;
        // ... 执行业务逻辑 ...
    }

    // ── 只读验签（非必选，推荐） ──
    function verify(
        Note calldata note,
        address expectedSigner,
        uint8 v,
        bytes32 r,
        bytes32 s
    ) external view returns (bool) {
        bytes32 digest = _hashTypedDataV4(hashNote(note));
        return ecrecover(digest, v, r, s) == expectedSigner;
    }
}
```

## 速查表：必须 vs 强烈建议

| 步骤 | 必要性 | 说明 |
|:---|:---|:---|
| ① 继承并初始化 EIP712 | ✅ **必须** | 父类需要 `name` 和 `version` 才能构造域分离器 |
| ② 定义 Struct | ✅ **必须** | 业务数据的载体，也是钱包展示给用户的内容 |
| ③ 定义 TypeHash | ✅ **必须** | EIP-712 标准强制要求，注意字符串拼写规则 |
| ④ 实现 hashStruct | ✅ **必须** | OZ 不知道你的结构体，必须自己编码 |
| ⑤ 生成 Digest | ✅ **必须** | 用 `_hashTypedDataV4` 是标准做法 |
| ⑥ 验签逻辑 | ✅ **必须** | 推荐用 `ECDSA.recover` 而非裸 `ecrecover` |
| ⑦ 防重放 | 🔴 **强烈建议** | 99% 场景都需要，否则签名可无限重放 |
| ⑧ 只读 verify 函数 | ⚡ **建议** | 便于中继器/后端预检查，节省 gas |

---

## EIP-712 的安全价值

| 安全属性 | 说明 |
|:---------|:-----|
| **钓鱼防御** | 钱包展示结构化数据，用户看清后再签 |
| **跨合约防御** | 签名绑到 `verifyingContract`，A 的签名不能在 B 上用 |
| **跨链防御** | 签名绑到 `chainId`，主网签名不能在 Sepolia 用 |
| **类型混淆防御** | TypeHash 唯一标识结构体形状 |
| **重放防御（同合约内）** | 由 nonce 机制提供，每个签名只能用一次 |

---

> **EIP712 帮你处理的是"签名生成"的标准和安全，而"签名使用"的安全（防重放）完全在你自己的代码中。**
