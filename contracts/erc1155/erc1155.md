## **ERC-1155：多代币标准（Multi Token Standard）**

- **一句话介绍**

  - 以太坊上为**同时管理多种代币类型**而制定的标准接口

  - 同一个合约中可以**混合包含同质化代币（如 ERC-20）、非同质化代币（如 ERC-721），以及其他混合形态（如半同质化代币）**

  - 核心思路是用一个合约管理 N 种代币，每种代币通过独立的 `id` 来区分

- **能做什么**

  - **一篮子管理**：一个合约即可管理无数种代币类型（同质化或非同质化均可），无需为每种代币单独部署合约，大幅节省链上字节码

  - **批量转账（核心亮点）**：支持一次交易同时转账**多种代币**（`safeBatchTransferFrom`），无论是同质化还是非同质化代币，均可打包处理，显著降低 gas 费用

  - **批量余额查询**：支持一次查询**多个地址**对**多种代币**的余额（`balanceOfBatch`），无需逐个查询

  - **授权管理**：采用“全量授权”模式（`setApprovalForAll`），一次性授权第三方管理某地址名下的**所有代币类型**，无需逐个代币授权

  - **安全转账**：提供 `safeTransferFrom` 和 `safeBatchTransferFrom`，转账时会检查接收方是否实现了 `ERC1155TokenReceiver` 接口，防止资产永久丢失

  - **元数据扩展**：可为每个 `id` 配置独立的元数据 URI（`URI` 事件），指向描述该代币名称、图片、属性等信息的 JSON 文件

- **适合出现在哪些场景**

  - **区块链游戏**：一个游戏合约可同时管理金币（同质化）、武器（非同质化）、皮肤（非同质化）、药水（同质化）等多种道具，无需为每种道具单独部署合约

  - **元宇宙与虚拟世界**：一块虚拟土地中可同时包含土地所有权（NFT）、土地上种植的作物数量（同质化代币）、建筑物（NFT）等

  - **电商与数字商品平台**：一个平台合约可管理不同品类的商品——限量款（NFT）和普通款（同质化代币），批量下单、批量发货一次交易完成

  - **DeFi 流动性池**：一个池子合约可同时管理多种资产的份额，批量存入或提取多种代币

  - **票务系统**：同一场活动可同时发行 VIP 票（NFT，每张座位不同）和普通票（同质化代币，大量相同），一个合约全部搞定

  - **批量空投**：项目方需要向大量地址发放多种代币时，ERC-1155 的批量转账功能可以一次性完成，省时省 gas

> **补充说明**：ERC-1155 的设计哲学是“大一统”——用一个合约解决所有代币类型的管理需求。它与 ERC-20 和 ERC-721 的核心区别在于：ERC-20 每个合约只管理一种同质化代币；ERC-721 每个合约只管理一种 NFT 集合；而 ERC-1155 一个合约可以同时管理 N 种同质化或非同质化代币，并通过批量操作大幅优化交易成本。
